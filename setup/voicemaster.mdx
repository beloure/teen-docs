# VoiceMaster

VoiceMaster lets members create temporary voice channels by joining a hub channel. Channels are automatically deleted when empty.

## How It Works

1. An admin sets up a VoiceMaster hub channel using `,vm setup`
2. When a member joins the hub, a personal voice channel is created
3. The owner controls it via an interactive button panel
4. When everyone leaves, the channel is deleted

## Setup

| Command | Description | Permission |
|---|---|---|
| `,vm setup` | Create a VoiceMaster hub channel | Manage Server |
| `,vm reset` | Remove the hub and config | Manage Server |
| `,vm category <category>` | Set the category for temp channels | Manage Server |

## Owner Controls

When a temp channel is created, the owner gets a panel with buttons:

- **Lock / Unlock** — prevent or allow others from joining
- **Rename** — change the channel name
- **User Limit** — set a max number of users
- **Hide / Reveal** — toggle visibility
- **Kick** — remove a user from the channel
- **Claim** — take ownership if the original creator left

## Admin Commands

| Command | Description | Permission |
|---|---|---|
| `,vm lock` | Lock a temp channel | Manage Channels |
| `,vm unlock` | Unlock a temp channel | Manage Channels |
| `,vm limit <number>` | Set user limit | Manage Channels |
| `,vm rename <name>` | Rename a temp channel | Manage Channels |
| `,vm permit <user>` | Allow a user into a locked channel | Channel Owner |
| `,vm reject <user>` | Block a user from the channel | Channel Owner |

Channels do not persist after the bot restarts or when all members leave.
